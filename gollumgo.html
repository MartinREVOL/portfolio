// BFS pathfinding — node-by-node AI movement

The enemy AI needs to find the optimal path to the player across the node graph, one node per turn. Rather than relying on UE5's built-in navmesh — which wasn't suited to our discrete grid structure — I built a custom BFS (Breadth-First Search) pathfinding system in Blueprints.

Each turn, the AI builds a search tree from its current node, exploring neighbors level by level until it finds the player's position. It then extracts the shortest path and moves one step along it. This guarantees the enemy always takes the optimal route, regardless of graph complexity.

Each AI runs its own independent BFS per turn — fully autonomous, no shared pathfinding state, easy to debug and extend.

// Multi-AI placement — no overlap on shared nodes

When multiple enemies land on the same node, they overlap visually. I built a spatial placement system that detects how many AIs occupy a given node, then calculates an offset for each one so they spread around the node center without clipping. The offset is recalculated every turn as enemies move.

// Grid data structure (shared)

I contributed to the grid data structure shared with the team: the node graph, connection table, and the runtime representation of which entities occupy which node at any given turn.
