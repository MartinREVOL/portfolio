// Enemy AI behavior — all enemy types

My primary responsibility on Onryō was designing and implementing the behavior systems for all enemy types in the game. Each enemy has its own C++ state machine that runs regardless of whether the enemy is AI-controlled or player-possessed.

The two main types — Base and Sentinel — share common exit conditions (death, stun) but differ in their core loop. The Base pursues the player actively: focus → move → shoot. The Sentinel moves to a fixed patrol spot first, then locks on and engages from range.

// Dual control source architecture

A key constraint: the same enemy Pawn must work both under AIController (normal gameplay) and PlayerController (when possessed by the player). I structured the AI state machines so that behavior logic stays entirely in the Pawn — meaning the same state machine runs correctly under both control sources without any branching. Possession is a controller swap, not a logic change.

// Shared stun state

All enemy types share a stun state implemented at the base class level — it can interrupt the behavior loop at any point and applies uniformly across all types.
